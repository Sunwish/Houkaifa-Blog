这看上去是在说废话，但是下面这段 JS 代码的意义和上面的 C# 代码是一样的，通过过程对照可以清楚的看明白 JS 中的面向对象表述逻辑以及 __proto__、prototype 和原型链。

首先，定义了一个函数 fun1()，但是注意，实际上 fun1() 不只是函数，它更是构造函数。此时可能会疑惑，此时连类都没有，哪里凭空定义的一个构造函数？

再往下看，fun1.prototype.prop1 = 123; 定义了 fun1 原型中的一个属性 prop1，其值为 123。也就是说，我们的 fun1 在定义之后就有了一个原型（prototype），而后的 fun1.prototype.method1 = function() { console.log("Hello World!"); }; 则进一步定义了这个原型除了属性 prop1 外还有一个方法 method1。

而在完成原型的这两条定义后，紧接着我们对 fun1 进行了 new 操作，得到了 fun1 的一个实例 fun1Instance，这也是为什么说 fun1 是构造函数的原因了。

在 C# 等面向对象语言中，我们的逻辑是先定义一个类，然后这个类拥有构造函数，属性和一些方法。而在 JS 中存在这样一种逻辑，我们是首先定义一个构造函数，然后藉由 prototype 这一属性，来丰富这个构造函数所能构造出来的实例。

具体看看 prototype 和 __proto__ 是什么

上节的这个 prototpye 我们称之为原型（只有函数才有），原型拥有两个属性，一个是 constructor，一个是 __proto__。

其中 constructor 顾名思义，就是构造函数，而这个构造函数将直接指回 fun1。这个不难理解，因为我们就是通过 fun1 的 fun1.prototye 来描绘原型，即是说这个原型就将由我们的 fun1 构造得出。

而 __prop__ 被称为隐式原型，因为它也是一个原型，只不过它不只是函数才有的，并且承担了 JS 中继承特性的发挥重任。

  • 对于对象而言，其隐式原型所指向的是构造出该对象的构造函数的显示原型，如 fun1Instance 对象的 __proto__ 就是 fun1.prototype。此处 fun1Instance 显然是从我们自己定义的原型构造得来的，因此上述结论很显然，而对于 JS 中的几种特殊的原生对象，如函数和数组，其默认的隐式原型就是由 JS 定义的内置原型，如所有函数的隐式原型是 Function.prototype，数组的隐式原型是 Array.prototype；

  • 对于原型而言，其隐式原型将直接指向其所处原型链上的上一级原型。如 fun1.prototype 的隐式原型就是 Object.prototype。这是因为 fun1 是函数，函数在原型链的上一级就是 Object。类似的，数组的上一级也是 Object。也就是说本文所给例子的原型链长这样：

而隐式原型之所以叫“隐式”，正是因为其在访问前面的原型链中原型的数据和方法时不需要显式地写出来。如 fun1Instance 是由 fun1 构造而来的，因此 fun1Instance 的隐式原型（fun1Instance.__proto__）是 fun1.prototpye，而 fun1Instance 要访问隐式原型的原型方法 method1 时并不需要这样写 fun1Instance.__proto__.method1()，而是直接写 fun1Instance.method1()即可。

由于前面提到，原型的隐式原型直接指向其所处原型链上的上一级原型，也就是说隐式原型层层嵌套传递，藉由隐式原型的这种传递，且可以省略 __proto__ 不写，我们就能够通过一个 . 运算符遍历到原型链上的所有下级原型数据和方法，实现了继承的效果。

做一个中场小结思考

经过上文梳理，我们可以明确：

  • (显式)原型类似于其它面向对象语言中的类的概念，但 JS 中一切皆对象，即使是简单的定义一个函数，这个函数也有其“类”（原型），且这个函数就正是其对应原型的构造函数。在 JS 中通过属性 prototype 来刻画原型，且这个属性只有函数才有，因为只有函数（不考虑 class 关键字）作为构造函数才有构造出原型的能力；
  • 隐式原型即 __prop__，该属性的访问不需要显式地写出。与函数才有原型不同的是，一切对象都有隐式原型。隐式原型可以被想作是一个指针，(1) 实例的隐式原型就是指向其构造函数的原型，(2) 而原型的隐式原型则直接指向原型链的上一级原型，由 (1)(2) 结合即可实现本级到下级所有原型链的贯通，实现继承特性。

思考如下代码，其为什么能输出元素 2 在数组 arr 中的下标？

这是因为在 JS 中数组也是对象，那么 arr 即是数组的一个实例，var arr = [1,2,3] 其实只是 var arr = new Array(1,2,3) 的一个简洁写法。既然是实例就有其实例的隐式原型。数组实例的隐式原型即是数组的原型，即 Array.prototype，不难想到，JavaScript 在Array.prototype 中内置了 indexOf() 函数。因此在代码执行的过程中，实际执行的其实相当于 arr.__proto__.indexOf(2)。

进一步地，设想我们定义了一个原型 MyArray 继承了 Array，那么我们的 MyArray 实例 myArr 将也可以使用 indexOf() 函数，因为 myArr.__proto__ 指向原型 MyArray.prototype，虽然其中我们没有定义 indexOf()，但是程序将进一步向下搜索 myArr.__proto__.__proto__，即 Array.prototype，其中就有 indexOf() 的定义了。

下面是一张非常棒的有助于进一步强化理解和梳理的可视化原型辅助理解图，可以注意到，只有函数有显式原型，且其原型的构造函数指回函数本身；实例的隐式原型即是其构造函数的原型，原型的隐式原型即是其原型链上的下一级原型，并最终指向 null。
