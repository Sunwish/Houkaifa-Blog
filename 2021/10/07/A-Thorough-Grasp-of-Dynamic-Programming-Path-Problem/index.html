一题吃透动态规划路径问题

概述

从一道 LC 困难题理解与解决 DP 路径问题解题思路及时空优化手段。

🔴 1289. 下降路径最小和 II

给你一个整数方阵 arr ，定义「非零偏移下降路径」为：从 arr 数组中的每一行选择一个数字，且按顺序选出来的数字中，相邻数字不在原数组的同一列。

请你返回非零偏移下降路径数字和的最小值。

示例 1：

提示：

  • 1 <= arr.length == arr[i].length <= 200
  • -99 <= arr[i][j] <= 99

题解

一、入门 DP 思路

开门见山，先了解一下DP 路径问题的初级解题思路：

  • 我们设置并维护一个二维数组 dp（或其它维，取决于问题），用于存储将路径中每一个可达节点作为题目要求目标的答案。如本题要求到达底层的最短路径长度，那么这个二维数组就将存储到达每一个节点的最短路径长度。此为状态定义。
  • 我们初始化这个 dp 数组，将不需要计算的起始数据填充。如本题第一行的所有节点到达自己的最短路径长度的就是自身代价，不需要计算。此为状态初始化。
  • 我们遍历每一个节点，并对每一个节点遍历可能的转移源，按题目要求留下需要的值作为该节点的 dp 值。如本题 dp 存储的是到达每一个节点的最短路径，那么对遍历过程中的任意节点，都遍历其可能的转移源，即除了正上方节点外的所有上层节点，然后留下到达自身最短的路径值作为当前节点的 dp 值。此为状态转移。

经过上面的步骤和后，将得到一张 n 维的 dp 表，存储了到达任意位置的题求答案，再根据题目具体要求，从表取值即可。如本题要求到达底层的最短路径长度，那么只需从表中底层的 dp 中返回最小的那个即可。

首先可以依照该思路解之，不难明确：

  • 状态定义：dp[i][j] 表示移动到位置 (i,j) 的最小路径和，且显然初始条件为 dp[0][:] = grid[0][:]，表示第一行各节点到自己的最短路径和即为自己本身。
  • 状态转移：由题意可知，dp[i][j] 可由上行除了正上方之的任意节点转移来，因此选择上层除正上方外的最小的那个 DP 节点进行转移即可。

三、优化时间复杂度

可以注意到，由于转移来源范围太广了（上行中除正上方节点外的所有节点），导致算法的时间复杂度直接飙到了 $O(n^3)$，虽然能 AC，但实在不够优雅。下面来尝试降低时间复杂度。

其实很容易发现，下一层将使用到的 dp 在遍历当前层的时候就可以被基本确定下来，因为下层 dp 期待的是由上层最小的 dp 转移来，那么只需要记录当前层最小的两个 dp 值供下层用就行了。

为什么是两个呢？Obviously，最小的那一个可能恰好处于下层遍历节点的正上方，这样的话最小的上层 dp 就不能用了，而如果最小的在正上方，那么第二小的就一定不在正上方，因此转移到下层的 dp 必然是当前层最小的那两个 dp 值当中的一个。

由于下层找转移来源的时候不需要再遍历整个上一层了，时间复杂度也就降到了 $O(n^2)$，执行用时从 45ms 优化到了 4ms，效果还算不错。

四、进一步优化空间复杂度

仅仅这样就可以了吗？

可以看到，经过上面的优化后，我们实际只用到了每层最小的那两个 dp 而已，其它的数据都并没有用到，那么为什么还要白花 $O(n)$ 的空间复杂度去存储那些用不到的 dp 值呢？因此我们可以将优化后的“残垣断壁”清理一下了，只存储当前层最小的两个 dp，其它数据一概不要，这样直接将空间复杂度压到了 $O(1)$。

同时，尽管此举的目的是优化空间，但由于之前离散地访问二维数组，空间局部性很差，经过空间优化取消了数组直接用整型变量存储需要的最小和次小 dp 后，执行用时也从中受益，进一步大大降低到了 2ms。

五、小结

本题的 DP 思路和优化方法是全系通用的。

  • DP 路径思路：明确状态定义、状态转移，从最简单的初始情况向外扩散，将中途解填充满所有节点，最终得到答案。
  • 时间优化角度：尽量压缩转移源的范围，只从可能的节点考虑转移。
  • 空间优化角度：断舍离，只存储有效范围的 dp 数据。
