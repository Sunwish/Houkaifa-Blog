概述

virtual 关键字作为 C++ 多态特性的表现载体，在多态 base class 的析构当中对内存泄漏的避免具有相当重要的意义，但是与此同时却也存在一些难以窥探到的风险性，如果在开发过程中对 virtual 的使用稍有疏忽就很有可能走上弯路，甚至造成一些不必要的麻烦。这些麻烦轻则导致程序运行质量的降低，耗费了更多的内存，重则造成更加严重的问题，产生内存泄漏、影响程序的移植性。
